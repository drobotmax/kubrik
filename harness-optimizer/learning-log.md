# KUBRIK Pipeline Learning Log

> Append-only лог рефлексий после каждого прогона pipeline.
> Записывается автоматически в конце каждого `/kubrik-pipeline`.
> После 3-5 записей - запустить полный harness optimizer (Round 4+).

## Формат записи

```
## Run [N] - [дата] - [client-slug]

**Бриф:** [краткое описание: рынок, тип, бюджет]
**Pipeline version:** v1 / v2 (rlm)
**Wiki pages used:** [какие wiki-страницы прочитали агенты]

### Что сработало
- [конкретное наблюдение]

### Что не сработало / было исправлено
- [что validator отловил, что пришлось руками править]

### Паттерн (если виден)
- [если эта проблема повторяется из прогона в прогон]

### Wiki update needed?
- [ ] Новый concept / [ ] Обновить существующий / [ ] Новый comparison / [ ] Нет
```
