# 02. Avatar Segmentation (Lost Chapter) — KUBRIK Protocol

## Зачем
Не все клиенты одинаково прибыльны. Цель: фокус на top-20% клиентов, которые:
- дольше остаются,
- платят больше,
- лучше внедряют,
- дают качественные кейсы и рефералы.

## Протокол 4 шага

### Шаг 1. Опросить клиентов
Собираем по каждому:
- Кто: гео, тип девелопера, стадия проекта, размер команды.
- Бизнес-метрики до старта: текущие каналы, CPL/CPA, объем лидов, скорость обработки, ROMI.
- Цель покупки: что хотели изменить.
- Почему купили: триггер, какой контент/встреча повлияли, сколько касаний до сделки.

### Шаг 2. Отсортировать top-20%
Критерии приоритета:
- прибыльность,
- скорость оплаты,
- ретеншн,
- минимальный операционный шум,
- готовность давать данные и внедрять рекомендации.

### Шаг 3. Найти общие признаки
Вывести 3-5 квалификаторов лучшего клиента.
Пример формата:
- Сегмент: городской девелопер с активным отделом продаж.
- Минимум трафика: уже тратят рекламный бюджет регулярно.
- Процессы: есть ответственный маркетинг owner.
- Мотивация: нужна быстрая оптимизация CPL + прозрачность.

### Шаг 4. Перестроить acquisition
- Маркетинг и sales говорят только на языке выбранного аватара.
- Нефит-клиентов мягко отсекаем заранее.
- В sales-процессе принудительно встраиваем путь, который прошли лучшие клиенты (правильные материалы, call-структура, proof).

## KPI для контроля
- Доля fit-лидов в pipeline
- Win rate по fit-сегменту
- LTV/CAC
- Gross margin по сегментам
- Время до first measurable result

## Решение для KUBRIK на старте
До накопления статистики использовать рабочую гипотезу ICP:
- Девелопер/проект с уже работающим маркетингом,
- есть бюджет и запрос на системность,
- готовы на weekly data review,
- решение принимается быстро (<= 30 дней).

Все новые лиды валидировать против этой гипотезы.
