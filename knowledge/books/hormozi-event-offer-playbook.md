# 03. Event Offer Playbook — KUBRIK

Цель: собирать авенты (вебинары, разборы, strategy sessions, demo days), которые приводят платежеспособных клиентов, а не просто регистрации.

## Шаблон проектирования авента (перед запуском)

### 1. Dream Outcome авента
Что человек должен захотеть получить после участия?
Пример:
- «Понять, где теряются 20-40% маркет-бюджета в девелоперке»
- «Получить roadmap роста лидов без расширения команды»

### 2. Проблемы/возражения (минимум 10)
Обязательно выписать заранее:
- нет времени,
- нет доверия к AI,
- сомнение в применимости к их проекту,
- страх сложного внедрения,
- страх “очередного подрядчика без результата”.

### 3. Решения для каждого возражения
На каждое возражение — конкретный ответ в структуре авента:
- кейс,
- демонстрация,
- мини-аудит live,
- чеклист,
- сценарий внедрения на 14/30/60 дней.

### 4. Delivery format (Trim & Stack)
Собрать стек авента из high-value компонентов:
- pre-event диагностическая анкета,
- 45-60 минут content с разбором,
- 15 минут framework под принятие решения,
- 1-page план внедрения,
- post-event follow-up sequence.

### 5. Offer Stack в конце авента
Показывать не «тариф», а итоговый пакет ценности:
- Диагностика + стратегическая карта
- План 30 дней
- Настройка первых AI-процессов
- Dashboard + weekly cadence
- Поддержка внедрения

### 6. Risk Reversal
Рекомендация: conditional service guarantee.
Пример формулировки для теста:
- «Если вы выполняете согласованный внедренческий чеклист, а KPI первого этапа не сдвигаются, мы продолжаем работу за свой счет до достижения agreed milestone».

### 7. Scarcity + Urgency
Этичная механика:
- лимит мест на внедрение в когорте,
- фиксированный дедлайн на бонус onboarding,
- следующий набор через N недель.

## Формула сильного названия авента
M-A-G-I-C (адаптация):
- Magnetic reason why
- Avatar
- Goal
- Interval
- Container

Примеры для KUBRIK:
- «Q2 Девелопер Growth Sprint: +системность маркетинга за 21 день»
- «Moscow Developer Leadflow Reset: 14-дневный диагностический интенсив»
- «AI Marketing Control Room для девелопера: 30-дневный launch-bootcamp»

## Мини-чеклист перед публикацией
- Понятен outcome в 1 фразу
- Есть сегмент (для кого/не для кого)
- Есть deadline
- Есть следующая ступень (CTA)
- Есть risk reversal
- Есть follow-up sequence на 7 дней
