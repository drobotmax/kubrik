# Signals Gateway: настройка источников, мест назначения и потоков данных
Настройте источники, места назначения и потоки данных в Signals Gateway, чтобы эффективно направлять свои данные. Используйте фильтры, чтобы персонализировать работу аккаунта и оптимизировать передачу данных.
##  Начало работы 
После начальной настройки войдите в свой аккаунт Signals Gateway. Вы можете сами [настроить Signals Gateway](https://developers.facebook.com/docs/marketing-api/gateway-products/signals-gateway/setup) либо поручить это разработчикам или партнерам вашей организации.
##  О потоках данных 
Поток данных — это система, которая перемещает информацию из одного места в другое. В Signals Gateway потоки соединяют ваши источники данных с выбранными местами назначения для бесперебойной передачи данных.
  1. Выполните вход в Signals Gateway.
  2. В меню выберите **Data pipelines** ("Потоки данных") и нажмите **Create data pipeline** ("Создать поток данных").
  3. Выберите нужный тип потока:
     * **Поток Signals Gateway**. Управляйте собственными данными, направляя их из своих источников в выбранные места назначения.
     * [Поток шлюза Conversions API](https://www.facebook.com/business/help/2041148702652965?id=818859032317965). Собирайте данные о веб-событиях с помощью существующего пикселя Meta и отправляйте их в набор данных Meta (если включен шлюз Conversions API).
  4. Выберите нужные источники данных, например [**Signals Gateway Pixel**](https://www.facebook.com/business/help/514664901027990?id=921478266803729) или **File upload** ("Загрузка файла"). 
  5. Выберите места назначения данных в вашем потоке: 
     * **Meta**. Отправляйте события в Meta через коннектор Meta Conversions API (если включен плагин Conversions API).
     * **Пользовательский коннектор**. Отправляйте события в уникальную конечную точку (только для потоков Signals Gateway).
  6. Самостоятельно выполните инструкции или отправьте их своему разработчику, чтобы настроить **Signals Gateway Pixel** или **загрузку файла**. 
  7. Настройте места назначения данных, следуя инструкциям на экране. 


##  Стратегии фильтрации 
Если вы добавите в поток данных фильтры, то события определенных типов перестанут попадать в ваши места назначения. Чтобы ограничить типы событий только для некоторых мест назначения, [создайте отдельные фильтры для каждого из этих мест](https://developers.facebook.com/docs/marketing-api/gateway-products/signals-gateway/event-filtering).
Узнать о фильтрации событий можно в нашей [документации](https://developers.facebook.com/docs/marketing-api/gateway-products/signals-gateway/event-filtering).
##  Глоссарий 
  * **Поток данных** — система, которая перемещает информацию из одного места в другое.
  * **Источник данных** — отправная точка данных, например сайт или приложение.
  * **Место назначения данных** — конечная точка, куда вы отправляете данные, например база данных или аналитическая платформа.
  * **Фильтрация** — процесс исключения из потока определенных точек данных, событий и доменов.
  * **Плагин** — компонент программного обеспечения, который добавляет в Signals Gateway определенные функции.
  * **Коннектор** — инструмент, связывающий поток Signals Gateway с удаленным местом назначения данных.
  * **Пиксель Meta** — небольшой фрагмент кода на сайте, собирающий данные для рекламной платформы Meta.


Узнать больше о конфигурации Signals Gateway можно в [документации для разработчиков](https://developers.facebook.com/docs/marketing-api/gateway-products/signals-gateway).
