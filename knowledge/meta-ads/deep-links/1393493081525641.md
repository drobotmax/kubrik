# О Signals Gateway
Signals Gateway — это облачная платформа, которая помогает эффективнее управлять и делиться данными. С ее помощью можно легко собирать, обрабатывать и передавать данные, сохраняя полный контроль над ними.
Главные преимущества Signals Gateway:
  * **Сохраняйте полный контроль над данными**. Собирайте данные из разных источников, в том числе сайтов, мобильных приложений и офлайн-каналов. [Блокируйте или отфильтровывайте определенные события](https://developers.facebook.com/docs/marketing-api/gateway-products/signals-gateway/event-filtering) и/или домены, чтобы соблюдать нормативные требования к конфиденциальности или достигать своих стратегических целей.
  * Передавайте данные в разные места назначения, например [Google BigQuery](https://developers.facebook.com/docs/marketing-api/gateway-products/signals-gateway/google-bigquery-connection-setup) и [Meta Conversions API](https://www.facebook.com/business/help/2041148702652965?id=818859032317965).
  * **Получайте расширенные возможности аналитики и отчетности**. Отправляйте данные в [Google BigQuery](https://developers.facebook.com/docs/marketing-api/gateway-products/signals-gateway/google-bigquery-connection-setup) или свои хранилища данных для детального анализа и получайте расширенную аналитику с ценными сведениями.
  * Используйте пользовательские коннекторы для простой интеграции с другими платформами.


##  Использование потоков данных в Signals Gateway 
С помощью потоков данных Signals Gateway централизует статистику из разных источников, например [Signals Gateway Pixel](https://www.facebook.com/business/help/514664901027990?id=921478266803729), загрузок офлайн-данных и интеграций CRM. Это позволяет собирать, обрабатывать и передавать данные на одной платформе, что упрощает управление сигналами и их анализ.

##  Источники данных и инструменты сбора информации 
Signals Gateway поддерживает разные источники и инструменты сбора данных, в том числе следующие:
  * [Signals Gateway Pixel](https://www.facebook.com/business/help/514664901027990?id=921478266803729). Собирает качественные события с ваших сайтов.
  * [Загрузка файла](https://www.facebook.com/business/help/1747311705837238?id=921478266803729). Позволяет загружать пользовательские офлайн-данные в вашу облачную среду по расписанию согласно вашим бизнес-потребностям.


##  Передача данных в другие места размещения 
Signals Gateway позволяет передавать данные:
  * **В места назначения вне Meta**. Отправляйте данные во внутренние системы, хранилища данных, рекламные сети или на аналитические платформы с помощью [пользовательских коннекторов](https://www.facebook.com/business/help/982767107037525). Такой гибкий подход обеспечивает полный контроль над способом и интервалом передачи данных в соответствии с вашей маркетинговой инфраструктурой.
  * [**В Google BigQuery**](https://developers.facebook.com/docs/marketing-api/gateway-products/signals-gateway/google-bigquery-connection-setup). Включите передачу данных в Google BigQuery для аналитики.
  * **[В Meta Conversions API](https://www.facebook.com/business/help/2041148702652965?id=818859032317965)**. Включите передачу данных в Meta Conversions API для более точного таргетинга рекламы и эффективной оптимизации. **[Conversions API](https://www.facebook.com/business/help/2041148702652965?id=818859032317965)** во время настройки потока данных.


##  Отличия от шлюза Conversions API 
Signals Gateway отличается от [шлюза Conversions API](https://www.facebook.com/business/help/387152639648383?id=818859032317965) по нескольким критериям, в том числе следующим:
  * **Источники данных**. Signals Gateway поддерживает несколько источников данных, тогда как шлюз Conversions API — только веб-события.
  * **Возможности передачи данных**. Signals Gateway позволяет передавать данные в несколько мест назначения. Meta получает доступ к этим данным, только если вы настроили поток данных в Meta Conversions API. Шлюз Conversions API передает данные только в Meta Conversions API. 
  * **Дополнительные функции**. В отличие от шлюза Conversions API, Signals Gateway предоставляет расширенные возможности аналитики и отчетности.

### Чем Signals Gateway отличается от шлюза Conversions API?
Signals Gateway — облачная платформа, которая позволяет собирать и обрабатывать данные, а также передавать их в несколько мест назначения. [Шлюз Conversions API](https://www.facebook.com/business/help/387152639648383?id=818859032317965) — это особый тип подключения, с помощью которого данные можно передавать в [Meta Conversions API](https://www.facebook.com/business/help/2041148702652965?id=818859032317965). Платформа Signals Gateway работает независимо от системы Meta, поэтому позволяет использовать любые источники данных и распределять сигналы по разным местам назначения.
![](https://scontent-prg1-1.xx.fbcdn.net/v/t39.8562-6/146597050_455534749152435_1802453867954835002_n.svg?_nc_cat=1&ccb=1-7&_nc_sid=f537c7&_nc_ohc=aXVV8xvJ_GcQ7kNvwHlDc6r&_nc_oc=AdpxRCyIaZgWtCiyYVpouqc9iu5YqUDJMwrDTBJEFNEJZUEPEvWvXOLbI_2mhUNO9Is&_nc_zt=14&_nc_ht=scontent-prg1-1.xx&_nc_gid=-r_pTsHIWzbQneQbCtAj5Q&_nc_ss=7a389&oh=00_AfykpWQfOd_y_0hOgIzXDhhu9d8mVLEj2o8Brn-lb2AmhA&oe=69CF18D8)
### Чем пиксель Meta отличается от Signals Gateway Pixel?
[Signals Gateway Pixel](https://www.facebook.com/business/help/514664901027990?id=921478266803729) собирает данные с вашего сайта и передает их исключительно на ваш облачный сервер Signals Gateway без помощи сторонних файлов cookie и независимо от серверов Meta. Это позволяет снизить влияние блокировщиков рекламы.
![](https://scontent-prg1-1.xx.fbcdn.net/v/t39.8562-6/146597050_455534749152435_1802453867954835002_n.svg?_nc_cat=1&ccb=1-7&_nc_sid=f537c7&_nc_ohc=aXVV8xvJ_GcQ7kNvwHlDc6r&_nc_oc=AdpxRCyIaZgWtCiyYVpouqc9iu5YqUDJMwrDTBJEFNEJZUEPEvWvXOLbI_2mhUNO9Is&_nc_zt=14&_nc_ht=scontent-prg1-1.xx&_nc_gid=-r_pTsHIWzbQneQbCtAj5Q&_nc_ss=7a389&oh=00_AfykpWQfOd_y_0hOgIzXDhhu9d8mVLEj2o8Brn-lb2AmhA&oe=69CF18D8)
### Signals Gateway можно использовать с существующей конфигурацией шлюза Conversions API?
Да. Signals Gateway можно использовать с существующей конфигурацией [шлюза Conversions API](https://www.facebook.com/business/help/387152639648383?id=818859032317965). При этом Signals Gateway предлагает дополнительные функции и возможности, недоступные со шлюзом Conversions API.
![](https://scontent-prg1-1.xx.fbcdn.net/v/t39.8562-6/146597050_455534749152435_1802453867954835002_n.svg?_nc_cat=1&ccb=1-7&_nc_sid=f537c7&_nc_ohc=aXVV8xvJ_GcQ7kNvwHlDc6r&_nc_oc=AdpxRCyIaZgWtCiyYVpouqc9iu5YqUDJMwrDTBJEFNEJZUEPEvWvXOLbI_2mhUNO9Is&_nc_zt=14&_nc_ht=scontent-prg1-1.xx&_nc_gid=-r_pTsHIWzbQneQbCtAj5Q&_nc_ss=7a389&oh=00_AfykpWQfOd_y_0hOgIzXDhhu9d8mVLEj2o8Brn-lb2AmhA&oe=69CF18D8)
### Как начать работу с Signals Gateway?
Для начала работы с Signals Gateway требуется:
  1. Настроить новый экземпляр в системе собственного поставщика облачных услуг или разместить его на партнерской облачной платформе. 
  2. Получить доступ к своей среде Signals Gateway.
  3. Настроить [источники данных и места назначения](https://www.facebook.com/business/help/1137792614422111?id=921478266803729).


[Обзор всего процесса](https://www.facebook.com/business/help/1834543500714664?id=921478266803729) и пошаговые инструкции можно найти в нашей [документации](https://developers.facebook.com/docs/marketing-api/gateway-products/signals-gateway/setup).
