# Проверка событий сервера с помощью инструмента тестирования событий
![](https://static.xx.fbcdn.net/mci_ab/public/cms/?ab_b=e&ab_page=CMS&ab_entry=438223955063449&version=1666716831&transcode_extension=webp)
В этой статье описывается тестирование событий на сайте, в приложении и событий CRM-системы, отправляемых через Conversions API. Узнать, как тестировать события, отправляемые через пиксель Meta, Facebook SDK, решение партнера по мобильным измерениям или App Events API, можно [_здесь_](https://www.facebook.com/business/help/2040882565969969).
Настроив серверные события на сайте, в приложении или события CRM-системы с помощью Conversions API, используйте инструмент тестирования событий в Meta Events Manager, чтобы:
  * Убедиться, что все полезные данные событий сервера настроены правильно и мы их получаем.
  * Выполнить отладку в случае подозрительных действий.


## Проверка событий сервера с помощью инструмента тестирования событий
Чтобы проверить события сервера с помощью инструмента тестирования событий:
  1. Откройте [**Events Manager**](https://business.facebook.com/events_manager2/list).
  2. Нажмите на вкладку **Источники данных** ![](https://static.xx.fbcdn.net/mci_ab/public/cms/?ab_b=e&ab_page=CMS&ab_entry=408108114296304&version=1774528167&transcode_extension=webp) в левой части экрана.
  3. Выберите название и ID нужных данных.
  4. Откройте вкладку **Тестирование событий**.
  5. Перейдите в раздел "Тестирование событий сервера".
  6. Если на шаге 3 вы выбрали набор данных, укажите, какие события вы хотите протестировать — на сайте или в приложении.
  7. Откройте терминал или нажмите **Graph API Explorer**. 
  8. Нажмите **Отправить**.


## Проверка событий сервера для интеграции CRM-системы с помощью инструмента тестирования событий
Чтобы проверить события сервера для интеграции CRM-системы с помощью инструмента тестирования событий:
  1. Откройте [**Events Manager**](https://business.facebook.com/events_manager2/list).
  2. Нажмите на вкладку **Источники данных** в левой части экрана.
  3. Выберите Страницу Facebook, подключенную к вашей CRM-системе.
  4. Выберите моментальную форму, которую используете для показа рекламы для сбора лидов.
  5. Нажмите **Сгенерировать новый лид**.
  6. Убедитесь, что подтверждаемый лид есть в вашей CRM-системе. Если такого лида нет в вашей CRM-системе, вам следует [проверить интеграцию лидов](https://www.facebook.com/business/help/734933888443065).
  7. Обновите статус лида в CRM-системе, чтобы инициировать событие.
  8. Событие CRM-системы о статусе подтверждения лида должно появиться в инструменте **Тестирование событий**. Например, если статус лида изменился с **Исходный** на **Возможность** , в инструменте должно появиться событие CRM-системы **Возможность**.


Тестовые события должны появиться в инструменте тестирования событий вскоре после отправки. Если события не появятся, проверьте структуру своих полезных данных. Вы можете проверить настройки, воспользовавшись [Помощником по полезным данным](https://developers.facebook.com/docs/marketing-api/facebook-pixel/server-side-api/payload-helper). Подробнее о том, как проверить правильность настройки данных на сервере, можно узнать на сайте [Meta for Developers](https://developers.facebook.com/docs/marketing-api/facebook-pixel/server-side-api#verify).
Тестовая информация хранится в инструменте тестирования событий в течение 24 часов или до тех пор, пока вы не очистите ее, нажав **Удалить действия**.
