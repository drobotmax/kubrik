# Рекомендации по работе с данными об офлайн-событиях
Данные об офлайн-событиях помогают оценить влияние вашей рекламы на количество офлайн-событий. Эту информацию можно использовать для оптимизации будущих кампаний. Ниже приведены рекомендации по подготовке и загрузке данных об офлайн-событиях.
  * **Загружайте данные об офлайн-событиях с точными метками времени и указывайте реальные суммы транзакций**. Метки времени, указанные с точностью до минут и секунд, позволяют загрузить данные о каждом уникальном событии, например о нескольких покупках, которые совершил один человек за день. Указывайте реальные суммы транзакций, чтобы в отчетах были представлены точные значения событий. Вы можете посмотреть совокупную денежную ценность загруженных, сопоставленных и отнесенных к вашей рекламе событий, чтобы лучше понять, насколько окупаются ваши затраты на рекламу.
  * **Указывайте в поле "ID заказа" уникальные для каждой транзакции личные данные**. Это поможет выполнить дедупликацию офлайн-событий и улучшить атрибуцию. С каждым ID заказа должно быть связано время уникального события и клиент. 
  * **Примечание**. Для каждого заказа можно отправлять в Meta только одно событие. Иначе могут возникнуть неточности в отчетности и оптимизации.

  * **Регулярно загружайте данные об офлайн-событиях в хронологическом порядке**. Так в отчетности будет меньше пробелов. Мы рекомендуем выполнять загрузку каждый день. Это позволит более эффективно оценить влияние рекламы на офлайн-события. Загружаемые данные обрабатываются в режиме реального времени, поэтому вы можете увидеть результаты сразу после добавления.
  * **Загружайте новые данные только о тех событиях, которые произошли после последней загрузки**. Мы сопоставляем показы, которые были зарегистрированы в течение последних 90 дней, с событиями, которые произошли через 1, 7 и 28 дней после показа. Если вы создали набор данных, но не загружали данные в течение 90 дней, у вас останется только 35 дней для загрузки контента. Если вы добавите данные в течение 35 дней, то у вас снова будет 90 дней, чтобы выполнить следующую загрузку. Рекомендуем загружать данные как можно чаще (желательно каждый день). Ранее загруженные события нельзя удалить. Наша система может только выполнить [дедупликацию](https://www.facebook.com/business/help/1772588746090250) событий за определенный период времени.
