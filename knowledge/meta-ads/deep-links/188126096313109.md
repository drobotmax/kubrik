# О настройке SKAdNetwork от Apple в Meta Events Manager
![](https://static.xx.fbcdn.net/mci_ab/public/cms/?ab_b=e&ab_page=CMS&ab_entry=438223955063449&version=1666716831&transcode_extension=webp)
Поддержка SKAdNetwork 4.0 от Apple внедряется постепенно и пока может быть доступна не всем. 
Вы можете настроить SKAdNetwork от Apple, чтобы проводить кампании для iOS 14+, направленные на продвижение приложений. Система атрибуции Meta совместима со SKAdNetwork и использует предоставляемую Apple информацию о конверсиях, чтобы создавать сводные отчеты по атрибуции для мобильных приложений. В Meta Events Manager доступен конфигуратор. С его помощью вы можете настраивать события для SKAdNetwork и продолжать измерять агрегированные результаты кампаний для iOS 14+.
Meta Ads Manager поддерживает несколько версий SKAdNetwork. Если ваша реклама показывается в приложениях Meta с iOS SDK 16.1 или более поздними версиями людям, которые используют устройства с ОС iOS 16.1 и более поздними версиями, будет использоваться SKAdNetwork 4.0. Если ваша реклама показывается людям, использующим предыдущую версию iOS или приложений Meta, будет использоваться предыдущая версия SKAdNetwork. [Подробнее об отчетности для SKAdNetwork от Apple](https://www.facebook.com/business/help/584603712214119).
## Получение данных для предыдущих версий SKAdNetwork от Apple
Для начала [подготовьте интеграцию с приложением](https://www.facebook.com/business/help/2750680505215705), а затем [настройте события в приложении в Events Manager](https://www.facebook.com/business/help/670955636925518).
## Получение данных для SKAdNetwork 4.0 от Apple
Если вы используете решение партнера по мобильным измерениям:
Узнайте у партнера по мобильным измерениям (MMP), поддерживает ли его решение SKAdNetwork 4.0. Если да:
  1. [Подготовьте интеграцию с приложением для SKAdNetwork](https://www.facebook.com/business/help/2750680505215705).
  2. Настройте конфигурацию приложения на сайте своего партнера по мобильным измерениям.
  3. [Настройте SKAdNetwork в Events Manager](https://www.facebook.com/business/help/670955636925518).


Если вы используете Conversions API или App Events API:
  1. [Подготовьте интеграцию с приложением для SKAdNetwork](https://www.facebook.com/business/help/2750680505215705).
  2. Изучите документацию Apple о том, как настроить SKAdNetwork 4.0 в вашем приложении.
  3. [Настройте SKAdNetwork в Events Manager](https://www.facebook.com/business/help/670955636925518). 


Настроив SKAdNetwork, вы сможете получать события конверсий из SKAdNetwork через Ads Insights API или напрямую в SKAdNetwork API от Apple (помимо Meta Ads Manager).
Если вы используете Facebook SDK для iOS:
  1. [Подготовьте свою интеграцию с приложением](https://www.facebook.com/business/help/2750680505215705).
  2. [Настройте события в своем приложении в Events Manager](https://www.facebook.com/business/help/670955636925518).


В SKAdNetwork 4.0 появились грубые значения ценности конверсии, что может помочь улучшить отчетность. [Подробнее о точной и грубой ценности конверсий](https://www.facebook.com/business/help/291167833340094).
Если вы настроите события с точной ценностью конверсий для SKAdNetwork в Events Manager и будете использовать Facebook SDK для iOS, события с грубыми значениями ценности могут поддерживаться автоматически.
Определите, поддерживаются ли грубые значения ценности конверсий для SKAdNetwork 4.0 автоматически.
  * Если ваша конфигурация SKAdNetwork в Events Manager содержит не более 3 отдельных или объединенных событий в приложении с точной ценностью конверсий без событий с грубыми значениями ценности, для вашего приложения автоматически создаются грубые значения. При этом используются такие правила:


  * Если конфигурация содержит 3 отдельных или объединенных события с точной ценностью конверсий, мы автоматически настроим события с грубыми значениями. Событие с наивысшим приоритетом будет автоматически отмечено как высокое, второе по приоритетности событие — как среднее, а третье — как низкое.
  * Если конфигурация содержит 2 отдельных или объединенных события с точной ценностью конверсий, мы автоматически настроим события с грубыми значениями. Событие с наивысшим приоритетом автоматически отмечается как высокое, а второе по приоритетности событие — как среднее.
  * Если конфигурация содержит одно отдельное или объединенное событие с точной ценностью конверсии, мы автоматически настроим событие с грубыми значениями. Такое событие автоматически получает высокий приоритет.
  * Если ваша конфигурация SKAdNetwork в Events Manager включает в себя ни одного или больше 3 отдельных либо объединенных событий в приложении с точной ценностью конверсий, события с грубыми значениями ценности не создаются для приложения автоматически. 


**Примечание**. Если вы не можете использовать SKAdNetwork в качестве метода атрибуции кампаний, создавайте кампании для iOS 14+, направленные на продвижение приложений, с применением протокола "Измерение на основе агрегированных данных о событиях". При этом вы не будете получать отчеты SKAdNetwork. [Подробнее о протоколе](https://www.facebook.com/business/help/721422165168355).
