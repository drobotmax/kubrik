# Интеграция пикселя Meta и SDK для мобильных устройств для рекламы автосалонов
Интегрируйте пиксель Meta, Conversions API или Facebook SDK, чтобы показывать рекламу автосалона людям, которые посетили ваш сайт или мобильное приложение. Пиксель, Conversions API и SDK позволяют Meta определять, кому показывать вашу рекламу, на основе намерений, поведения и действий пользователей. С помощью этой информации Meta сможет предложить подходящие рекомендации потенциальным клиентам, которые с высокой вероятностью совершат конверсию. Например, можно показывать персонализированную рекламу людям, которые просмотрели конкретный автомобиль или ввели платежную информацию.
Из этого руководства вы узнаете, как: 
  * Настроить пиксель.
  * Изменить пиксель для рекламы ассортимента автосалона.
  * Изменить пиксель для рекламы моделей автомобилей.
  * Интегрировать SDK для мобильного приложения.


## Настройка пикселя Meta
Настройте пиксель Meta и добавьте его на свой сайт, чтобы создать рекламу ассортимента автосалона. 
  * Если вы рекламодатель, следуйте [руководству по созданию и установке пикселя Meta](https://www.facebook.com/business/m/pixel-setup-get-started).
  * Если вы разработчик, ознакомьтесь с документацией по настройке API на сайте [Meta for Developers](https://developers.facebook.com/docs/marketing-api/dynamic-ads-auto/auto-event). 


После настройки пикселя вы можете изменять его на ключевых страницах сайта в соответствии с целями вашей рекламы. В зависимости от того, что вы рекламируете (ассортимент автосалона или модели автомобилей), вам нужно настроить пиксель для разных событий.

### Изменение пикселя для рекламы ассортимента автосалона
В зависимости от бизнес-целей вы можете изменить свой пиксель и показывать рекламу людям на разных этапах воронки продаж с помощью специально настроенных событий. Для рекламы ассортимента автосалона необходимо добавить 4 специально настроенных события на разные страницы вашего сайта. Попросите разработчика сайта изменить пиксель для каждой из следующих страниц:
  1. **Search** — для отслеживания интереса к транспортным средствам.
  2. **ViewContent** — для отслеживания просмотров страницы сведений о транспортном средстве. 
  3. **AddToWishlist** — для отслеживания транспортных средств, добавленных в избранное, выделенных или отмеченных с намерением купить. 
  4. **Lead** — для отслеживания полученных лидов.
  5. **Purchase** — для отслеживания онлайн-покупок транспортных средств, их бронирования и предварительных заказов (необязательно).


При выборе событий укажите конкретные параметры, которые предоставляют полезную информацию, важную для вашей компании. В рекламе автосалонов используются события с набором **обязательных** параметров. Полный список параметров и другие требования к форматированию можно найти на сайте [Meta for Developers](https://developers.facebook.com/docs/marketing-api/dynamic-ads-auto/auto-event). 
**Стандартные события и параметры для рекламы ассортимента автосалона**
Стандартные события помогают Meta показывать вашу рекламу клиентам, которые с большой вероятностью совершат конверсию. В зависимости от типа ассортимента вы можете добавлять стандартные события с определенными параметрами, чтобы отслеживать, кто посещал ваш сайт, добавлял в избранное автомобиль, идентификатор и цену товара, а также многое другое.
**Примечание**. Параметры необязательно использовать для общей настройки пикселя Meta, однако реклама ассортимента автосалона требует определенных стандартных событий и параметров. Обязательно проверьте различные варианты написания своих параметров, обращая внимание также на регистр. Чтобы быстрее приступить к работе, ознакомьтесь со [стандартными событиями и параметрами](https://www.facebook.com/business/help/518588851979334) для рекламы ассортимента автосалона.
При добавлении параметров убедитесь, что параметр **vehicle_id** (ID транспортного средства) соответствует параметру **content_id** (ID контента), используемому при настройке пикселя на вашем сайте. Из-за различий в параметрах могут возникнуть неточности в отчетности и отслеживании конверсий. Также можно передавать по несколько ID с одной страницы путем отправки массива данных.

### Изменение пикселя для рекламы моделей автомобилей
Чтобы настроить ретаргетинг на клиентов на разных уровнях воронки продаж, нужно внести изменения в пиксель Meta на выбранных страницах. В дополнение к коду, установленному на всех страницах сайта, для рекламы моделей автомобилей вам потребуется добавить на разные страницы пиксель, который будет отслеживать 4 специально настроенных события. Попросите разработчика сайта изменить пиксель для каждой из следующих страниц:
  1. **Search** — для отслеживания поисковых запросов, связанных с предложениями автосалона.
  2. **ViewContent** — для отслеживания просмотров страниц с информацией о транспортных средствах, модельном ряде или специальных предложениях.
  3. **AddToWishlist** — для отслеживания предложений, которые люди добавили в список сохраненных или понравившихся, либо к которым проявили интерес.
  4. **Lead** — для отслеживания полученных лидов.
  5. **Purchase** — для отслеживания онлайн-покупок транспортных средств, их бронирования и предварительных заказов (необязательно).


При изменении событий пикселя укажите конкретные параметры, которые предоставляют полезную информацию, важную для вашей компании. 
**Примечание**. В рекламе моделей автомобилей используются события с набором обязательных параметров. Подробнее ознакомиться с разными параметрами и форматами, требуемыми для рекламы моделей автомобилей, можно на сайте [Meta for Developers](https://developers.facebook.com/docs/marketing-api/auto-ads/guides/aoa#install-pixel).
**Стандартные параметры пикселя для рекламы моделей автомобилей**
На каждой странице можно добавить стандартные события для отслеживания действий клиентов на сайте. Важно добавить эти стандартные события с максимальной точностью, без изменения написания или регистра. [Подробнее о стандартных событиях и параметрах для рекламы моделей автомобилей](https://developers.facebook.com/docs/marketing-api/auto-ads/guides/aoa#content-type).
Если вы захотите продвигать предложения, параметр **vehicle_offer_id** (ID предложения транспортного средства) должен соответствовать параметру **content_id** (ID контента), который используется в пикселе на вашем сайте или передается с помощью Conversions API, иначе могут возникнуть неточности в отчетности и отслеживании конверсий. Также можно отправлять по несколько ID товаров с одной страницы путем отправки массива данных.

## Интеграция Facebook SDK для мобильного приложения
**Начало работы с Facebook SDK**
Чтобы использовать Facebook SDK для iOS, Android или JavaScript (на веб-платформах), вам потребуется ID приложения Facebook. Если у вас уже есть [приложение](https://developers.facebook.com/apps), можете использовать его. Если нет, создайте его с помощью [этого руководства](https://developers.facebook.com/quickstarts). Если вам нужна помощь, следуйте [руководству по регистрации и конфигурации приложения](https://developers.facebook.com/docs/apps/register). 
Вам будет нужно добавить SDK в свое приложение. [Подробнее](https://developers.facebook.com/docs/app-ads/sdk). 
Прежде чем добавить SDK в приложение, ознакомьтесь с разделом ниже, чтобы также интегрировать события в приложении, необходимые для рекламы ассортимента автосалона. 
**Настройка событий в приложении на ключевых страницах нижних уровней воронки**
Для динамического ретаргетинга актуальных товаров на пользователей вашего приложения вам потребуется добавить в него те же 4 обязательных события.  
| **Событие**  | **Событие в приложении**  |  
| --- | --- |  
| Search  | fb_mobile_search  |  
| ViewContent  | fb_mobile_content_view  |  
| AddToWishlist  | fb_mobile_add_to_wishlist  |  
| Lead  | Lead  |  
Подробнее о том, [как добавлять специально настроенные события для рекламы моделей автомобилей](https://developers.facebook.com/docs/marketing-api/dynamic-ads-auto/auto-event).
