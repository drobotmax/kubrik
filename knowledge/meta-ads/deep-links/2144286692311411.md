# Локализация каталога с помощью фида для стран или языков
Фид для языков или стран — это особый тип фида данных, который вы создаете и загружаете в свой каталог. Фиды для стран и языков содержат локализованные версии данных о ваших товарах. Если в каталоге есть такая информация, в вашей рекламе и магазине каждому покупателю будут автоматически показываться нужные сведения. Например, жители США будут видеть описания товаров на английском языке, а цены — в долларах США, в то время как жителям Франции эта информация будет видна на французском языке с ценами в евро.
## Прежде чем начать
  * [Создайте каталог](https://www.facebook.com/business/help/1275400645914358) для языка и страны по умолчанию. Это должна быть страна, где вы ведете основную деятельность, и язык, на котором говорит большинство ваших клиентов. Не забудьте указать страну и язык по умолчанию в разделе **Локализация** на вкладке **Настройки** в Commerce Manager.
  * [Загрузите фид данных в каталог](https://www.facebook.com/business/help/125074381480892) с информацией о товарах для страны или языка по умолчанию. Это ваш главный фид. Фиды для других стран и языков будут его дополнять. Перед тем как запустить показ объявлений из каталога Advantage+, убедитесь, что ID контента каждого товара в фиде данных совпадает с ID контента в коде пикселя Meta на сайте.


## Скачивание шаблона фида для стран и языков
Чтобы создать фид для стран или языков, скачайте шаблон. Для этого:
  1. Перейдите в [Commerce Manager](https://www.facebook.com/commerce_manager) и выберите свой каталог.
  2. Откройте вкладку **Каталог** и нажмите **Источники данных**.
  3. В разделе **Фиды для стран и языков** выберите **Добавить новый фид**.
  4. Нажмите **Создать шаблон**. Во всплывающем окне выберите **Шаблон фида для стран** или **Шаблон фида для языков** и нажмите **Далее**. Чтобы локализовать данные о товаре для страны и языка, нужно выполнить эти шаги дважды и создать два отдельных шаблона. В фиды стран и языков можно включить несколько вариантов.
  5. Выберите страны или языки, которые хотите охватить. Нажмите **Далее**.
  6. Выберите поля со свойствами товаров, которые нужно локализовать. В фид для стран добавьте сведения о наличии, цены, статус, ссылки и размеры, а в фид для языков — названия и описания. В конструкторе шаблонов вам будут предложены стандартные поля. Вы также можете добавить другие поля, когда будете заполнять шаблон. Закончив, нажмите **Далее**.
  7. Нажмите **Скачать** , чтобы скачать шаблон в формате CSV. Нажмите **Закрыть**.


Вы скачали шаблон фида для стран или языков. Чтобы заполнить его, выполните указанные ниже действия.
## Заполнение шаблона фида для стран или языков
  1. Откройте скачанный файл CSV в программе для работы с электронными таблицами, например Microsoft Excel.
  2. Найдите в шаблоне поле **id**. В нем содержатся уникальные ID контента ваших товаров. Удалите товары, данные о которых не хотите локализовать. Укажите ID контента для товаров из каталога, которые хотите добавить. Каждый такой идентификатор в фиде для стран или языков должен совпадать с соответствующим ID контента в вашем главном фиде, чтобы мы могли сопоставить локализованные данные с правильной позицией в каталоге.
  3. Найдите поле **override**. В нем содержатся коды стран или языков, для которых вы предоставляете данные. Чтобы указать больше стран или языков, добавьте новую строку для каждого товара и введите код из соответствующего списка:

  4. В вашем шаблоне также будут указаны поля, которые вы выбрали в конструкторе шаблонов, например **price** или **description**. В каждом из этих полей введите локализованные данные о товаре для соответствующей страны или языка в поле **override**.
     * В зависимости от типа вашего ассортимента, заполните поля согласно требованиям для [товаров](https://developers.facebook.com/docs/marketing-api/catalog/reference/#da-commerce), [гостиниц](https://developers.facebook.com/docs/marketing-api/hotel-ads/catalog#supported-fields-hotel), [авиарейсов](https://developers.facebook.com/docs/marketing-api/flight-ads/catalog#supported-fields-flight), [туристических направлений](https://developers.facebook.com/docs/marketing-api/destination-ads/catalog#supported-fields-destination), [транспортных средств](https://developers.facebook.com/docs/marketing-api/dynamic-ads-auto/auto-catalog#vehicle) или [недвижимости](https://developers.facebook.com/docs/marketing-api/real-estate-ads/get-started#home-listing-fields). Названия полей и определенные поддерживаемые значения должны быть написаны на английском (США), как указано в требованиях.
     * Для цен добавляйте трехбуквенный [код валюты по стандарту ISO 4217](https://l.facebook.com/l.php?u=http%3A%2F%2Fwww.currency-iso.org%2Fen%2Fhome%2Ftables.html&h=AT5xC8U-PISMQMMYm6pWhGhJHIJCS1Ko9KpP6HhUB7sZE9Ggfy0WaF6T962E9w0_0p24kESELY2UJx7F7e5evhixpx-wkbYC2ads9FE8uvtDN1_lmsbbCZ0pl_ij62Q0WJCJRw3tqR-YFg), например EUR вместо символа €. В качестве десятичного разделителя используйте точку (.), а не запятую (,). Например, 5.99 EUR — правильный вариант, а 5,99 € — неправильный.

| 5678  | FR  | in stock  | 24.99 EUR  | http://www.jaspersmarket.com/fr/products/5678.html  |  
**Пример фида для языков** :

| 5678  | fr_XX  | Jupe Plissée  | Jupe plissée de longueur mi-mollet avec une ceinture.  |

### Поддерживаемые поля в фидах для стран и языков
**Примечание**. Поля Price, sale_price, unit_price, base_price, availability и status должны быть только в фиде для стран. Их нельзя добавить в фид для языков. Это нужно, чтобы люди видели правильные локализованные данные о товарах.
