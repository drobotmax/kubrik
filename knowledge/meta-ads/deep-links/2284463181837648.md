# О графике загрузки фида данных для каталога
Вы можете настроить график загрузок фида данных (ежечасно, ежедневно или еженедельно) и управлять им, чтобы информация в каталоге всегда была актуальной. Сохраните файл данных в хранилище и предоставьте нам ссылку. Извлечение файла и обновление каталога будут происходить по настроенному вами графику.
## Требования к фидам данных, обновляемым по графику
  * **Поддерживаемые форматы файлов** : CSV, TSV, Excel (XLSX), XML (RSS/ATOM) или Google Таблицы. Ссылка должна начинаться с http, https, ftp или sftp и вести к файлу, а не на ваш сайт, Страницу Facebook или в другое место.
  * **Максимальный размер файла** : 4 ГБ. Если ваш файл превышает допустимый размер или содержит более миллиона позиций, распределите их по нескольким фидам данных и загрузите по отдельности. Также для более быстрой обработки больших файлов во время загрузки фидов рекомендуется использовать параметр обновления (см. ниже) вместо параметра замены.
  * **Доступ к файлу по ссылке** : Если ваш файл защищен паролем, укажите имя пользователя и пароль. Если вы используете Google Таблицы, убедитесь, что для функции доступа по ссылке указано значение **Все, у кого есть ссылка** вместо **Доступ ограничен**. 


## Добавление и изменение графика загрузки фида
График можно указать при [загрузке нового фида данных](https://www.facebook.com/business/help/125074381480892) в Commerce Manager. Вы можете в любое время добавить или изменить график, нажав **Источники данных**. Для этого выберите фид данных, перейдите в его **настройки** и измените параметры в разделе **Графики**.
## Различия между обновлением и заменой фида данных после каждой загрузки
Для графика загрузки есть два параметра. Вы можете выбрать, что должно происходить с фидом при каждой загрузке: обновление или замена.
  * **График замены** — новый фид полностью заменяет всю информацию о товарах в предыдущем фиде, добавляя данные о новых товарах в каталог, обновляя о существующих и удаляя о тех, которых в новом файле нет. Это параметр по умолчанию, если вы [добавляете новый фид данных](https://www.facebook.com/business/help/125074381480892) по графику. Этот вариант загрузки займет больше времени.
  * **График обновления** — после быстрого сканирования вашего фида система добавит новые позиции в каталог и обновит существующие. Этот вариант загрузки займет меньше времени, однако удаленные позиции и поля будут игнорироваться. Чтобы удалить товары или данные с помощью параметра обновления: 
    * Чтобы [удалить товары](https://www.facebook.com/business/help/428079314773256), добавьте в фид поле **delete** и укажите значение **true** для каждой позиции. Для удаления товаров достаточно загрузить фид один раз. Вы сможете удалить товары из файла позже.
    * Чтобы убрать данные о товаре из определенного поля, оставьте это поле в фиде пустым. Например, чтобы удалить цену со скидкой, установите для параметра **sale_price** значение 0 или оставьте его пустым. Для удаления данных в этом поле достаточно загрузить фид один раз. Потом поле можно удалить.


## Советы по настройке графика загрузки фида
  * Запланируйте загрузку так, чтобы она происходила после того, как вы обычно меняете фид. Так в каталоге будут оперативно появляться актуальные данные о товарах. Например, если вы меняете фид каждый день в 9:00, укажите, что запланированные обновления должны происходить ежедневно в 9:15.
  * Если вы используете только параметр обновления, количество товаров в каталоге может расти, поскольку эта настройка не удаляет товары (если только вы не используете поле **delete**). Используйте оба параметра для графика загрузки, чтобы гарантировать актуальность данных о товарах. Например, вы можете настроить: 
    * Почасовой график для обновления данных о товарах, таких как цены и наличие.
    * Еженедельный или ежемесячный график замены для удаления устаревших позиций из каталога.
  * Между обработкой замены и обновления фида должно пройти не менее 10 минут.
