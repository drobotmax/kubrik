# Рекомендации по работе с Conversions API, которые помогут улучшить результативность рекламы
![](https://static.xx.fbcdn.net/mci_ab/public/cms/?ab_b=e&ab_page=CMS&ab_entry=438223955063449&version=1666716831&transcode_extension=webp)
Эта статья для компаний, использующих Conversions API для отправки событий на сайте. 
Эти советы по работе с [Conversions API](https://www.facebook.com/business/help/2041148702652965) помогут вам снизить цену за действие и тем самым улучшить результативность рекламы.
Чтобы повысить результативность рекламы на платформах Meta, воспользуйтесь этими рекомендациями при настройке Conversions API. Если вы уже настроили Conversions API, приведенные рекомендации помогут вам улучшить его работу.
## Отправка избыточных событий
  * **Используйте Conversions API вместе с пикселем Meta, чтобы отправлять события через оба инструмента**. Это называется конфигурацией с избыточными событиями. Например, если вы отправляете веб-события покупки, начала оформления заказа и контакта с помощью пикселя, отправляйте те же события сервера через Conversions API. Такой подход позволяет не упустить события, которые пиксель мог не зарегистрировать из-за проблем с соединением или ошибки загрузки страницы. Подробнее о том, [как проверить, отправляете ли вы избыточные веб-события](https://www.facebook.com/business/help/1541268312717919).
  * **Следите за равноценностью между интеграциями пикселя Meta и Conversions API, отслеживая покрытие событий**. Покрытие событий соответствует сумме событий браузера и сервера, отправленных в Meta. Оптимальный коэффициент покрытия событий от Conversions API и пикселя Meta для точной отчетности и оптимальной результативности рекламы — 75 %. Чтобы достичь его, отправляйте через Conversions API все необходимые события, в том числе те, которые могли быть не зарегистрированы вследствие ограничений браузера или технических проблем. Регулярный контроль отправки событий в Meta Events Manager поможет определить возможности для улучшения и оптимизировать конфигурацию. Подробнее о том, [как проверить покрытие событий в Meta Events Manager](https://www.facebook.com/business/help/1541268312717919).
  * **Дедуплицируйте избыточные события, чтобы уменьшить количество повторов**. Если вы используете и пиксель, и Conversions API, мы можем получить одинаковые события от браузера и сервера. С помощью дедупликации вы сможете оставить только одно событие, удалив его дубликаты. Для определения дубликатов необходимо указать определенные параметры. Подробнее о [дедупликации](https://www.facebook.com/business/help/823677331451951) и о том, [как отслеживать дедуплицированные события](https://www.facebook.com/business/help/1541268312717919).


## Качество сопоставления событий
  * **Улучшайте качество сопоставления событий** с параметрами информации о клиентах в аккаунте Meta. Более качественное сопоставление событий может увеличить [количество зарегистрированных дополнительных конверсий](https://www.facebook.com/business/help/453888373437795). В Events Manager приводятся рекомендации по улучшению сопоставления событий. Изучая тенденции в Meta Events Manager, можно понять, как со временем менялся коэффициент конверсии и были ли получены новые конверсии. Благодаря полученной статистике вы узнаете, как изменения в качестве данных влияет на коэффициент конверсии. Используйте эти данные, чтобы усовершенствовать стратегию и повысить результативность рекламы. Подробнее о [качестве сопоставления событий](https://www.facebook.com/business/help/765081237991954) и том, [как проверить его в Meta Events Manager](https://www.facebook.com/business/help/1541268312717919). 
  * **Указывайте параметры, чтобы улучшить качество сопоставления событий в Meta Events Manager**. 
    * **Передавайте больше параметров информации о клиентах**. Чем больше событий с этими параметрами вы отправляете, тем выше вероятность сопоставления. Подробнее о том, [как параметры информации о клиентах улучшают качество сопоставления событий](https://www.facebook.com/business/help/765081237991954). [руководством по получению согласия](https://developers.facebook.com/docs/privacy) и обратитесь к юристу.
    * **По возможности присваивайте высокий приоритет параметрам, которые с большей вероятностью улучшат качество сопоставления событий**. Целевая аудитория влияет на результативность параметров, но у каждого параметра есть приоритет. [Подробнее о параметрах информации о клиентах и об уровнях их приоритетности](https://www.facebook.com/business/help/765081237991954).


## Другие рекомендации
  * **Отправляйте события в режиме реального времени или как можно ближе к этому**. Так система показа рекламы сможет лучше оценить вероятность, с которой просмотревший вашу рекламу человек совершит нужное действие. Подробнее о [задержке в получении данных о событиях](https://www.facebook.com/business/help/379226453470947) и о том, [как проверить регулярность обновления данных о событиях сервера в Events Manager](https://www.facebook.com/business/help/1541268312717919).
  * **Улучшите интеграцию, подключив по возможности больше пикселей и отправляя больше событий**. Подключение несвязанных пикселей к интеграции Conversions API может повысить результативность рекламы, а отправка событий с помощью пикселя и Conversions API предоставит больше данных для ее оптимизации.
  * **Постоянно контролируйте работу Conversions API**. Регулярно отслеживая данные о событиях, вы убедитесь, что функции работают корректно, и выявите возможности для улучшения. Подробнее о том, [как контролировать работу Conversions API](https://www.facebook.com/business/help/586304118741779).
  * **Применяйте рекомендации инструмента "Оценка возможностей", если они вам доступны**. Соответствующие требованиям рекламодатели могут получать проверенные на практике рекомендации по видеорекламе для мобильных устройств в инструменте "Оценка возможностей". Это новый инструмент от Meta, который помогает определить и ранжировать эффективные рекомендации для кампаний практически в режиме реального времени. [Подробнее об оценке возможностей](https://www.facebook.com/business/help/804913634782260). 
**Примечание.** Оценка возможностей (в том числе высокая) сама по себе не отражает текущую или будущую результативность. Фактическая результативность зависит от многих факторов, поэтому оценка возможностей не гарантирует ее.
