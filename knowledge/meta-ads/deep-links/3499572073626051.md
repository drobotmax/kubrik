# Об отчетности о событиях в приложении, полученной через протокол "Измерение на основе агрегированных данных о событиях"
![](https://static.xx.fbcdn.net/mci_ab/public/cms/?ab_b=e&ab_page=CMS&ab_entry=438223955063449&version=1666716831&transcode_extension=webp)
С 9 октября 2024 года мы отправляем отчетность по Измерениям на основе агрегированных данных о событиях (AEM) партнерам по мобильным измерениям (MMP), чтобы создать комплексное представление о результативности мер по привлечению новых пользователей на платформах MMP, включенных в отчетность. Прежде чем использовать данные для измерения или показа рекламы, мы по-прежнему применяем методы защиты конфиденциальности, например удаление идентификаторов, добавление дифференциальной конфиденциальности и агрегацию данных пользователей. Мы сделали эту функцию доступной для всех партнеров MMP. Время интеграции может отличаться.
Если вы показываете кампании для продвижения приложения с использованием модели атрибуции Meta для устройств с iOS 14 и более поздними версиями (протоколом "Измерение на основе агрегированных данных о событиях"), то результативность системы оптимизируется, в том числе отчетность создается практически в режиме реального времени. Ниже описано, как работает отчетность, для которой используется такой метод атрибутирования кампаний.
## Преимущества отчетности по кампаниям для продвижения приложения
  * **Обеспечение отчетности практически в режиме реального времени**. Измерение на основе агрегированных данных о событиях происходит почти в реальном времени.
  * **Возможность дополнительного использования отчетности SKAdNetwork**. Чтобы получить отчетность SKAdNetwork по кампании для продвижения приложения, в которой используется протокол "Измерение на основе агрегированных данных о событиях", выберите **Сравнить настройки атрибуции** в раскрывающемся меню **Столбцы** и нажмите **Дополнительные параметры отчетности**. Выберите **View** или **Click** , чтобы просмотреть параметры SKAdNetwork на одном экране.  
  
**Примечание**. Отчетность протокола "Измерение на основе агрегированных данных о событиях" и данные SKAdNetwork [могут отличаться](https://www.facebook.com/business/help/1356268495231843).
  * **Параметры отчетности по умолчанию и сравнение настроек атрибуции**. Параметры отчетности протокола "Измерение на основе агрегированных данных о событиях" по умолчанию основаны на выбранном параметре атрибуции. Сейчас платформы Meta поддерживают модели атрибуции "1 день после клика", "7 дней после клика" и "28 дней после клика" для событий установки, а также "1 день после клика" и "7 дней после клика" для кампаний с оптимизацией для событий в приложении и оптимизацией для ценности. Для событий после установки доступна отчетность по разным окнам атрибуции, в том числе "1 день после клика", "7 дней после клика" и "28 дней после клика", в инструменте **Сравнение настроек атрибуции**.
  * **Статистическое моделирование**. Чтобы обеспечить комплексное представление о результатах при неполных или отсутствующих данных о событиях, может использоваться статистическое моделирование. К некоторым результатам и конверсиям в Ads Manager и Отчетах по рекламе добавлены [сноски](https://www.facebook.com/business/help/1329822420714248), в которых описаны особенности расчета.
  * **Разбивка данных**. У вас есть доступ к подробной статистике о результативности, а также следующим разбивкам: "Возраст", "Пол", "Возраст и пол", "Страна", "Устройство показов", "Платформа", "Платформа и устройство" и "Место размещения".


## Преимущества отчетности по кампаниям для повторного вовлечения в приложении
  * **Поддерживаемые события в отчетности**. Отчетность доступна для всех типов кампаний для повторного вовлечения (в том числе с целями "Продажи", "Лиды" и "Вовлеченность"), в которых в качестве места получения конверсий выбрано приложение. Кампании этих типов оптимизируются для событий в приложении.
  * **Статистическое моделирование**. Чтобы обеспечить комплексное представление о результатах при неполных или отсутствующих данных о событиях, мы можем использовать статистическое моделирование. Мы добавили [сноски](https://www.facebook.com/business/help/1329822420714248) к некоторым результатам и конверсиям в Ads Manager и Отчетах по рекламе. В них описаны особенности расчета.
  * **Разбивка данных**. У рекламодателей есть доступ к разбивкам для более подробной статистики о результативности. В кампаниях, в которых используется оптимизация для событий в приложении и протокол "Измерение на основе агрегированных данных о событиях", можно просмотреть 8 разбивок. Это "Возраст", "Пол", "Возраст и пол", "Страна", "Устройство показов", "Платформа", "Платформа и устройство" и "Место размещения".
