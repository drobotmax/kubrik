# Устранение проблем при несоответствии приложения требованиям протокола "Измерение на основе агрегированных данных о событиях"
Когда вы создаете рекламу в Ads Manager или [проверяете события в приложении на соответствие требованиям протокола "Измерение на основе агрегированных данных о событиях" в Events Manager](https://www.facebook.com/business/help/617021863580582), то можете увидеть сообщение о том, что ваше событие в приложении не отвечает требованиям, с объяснением причин.
Ниже приведен список некоторых сообщений и объяснение, что они значат и какие действия нужно предпринять вам или вашему разработчику, чтобы решить проблему с соответствием.
  * **Свяжитесь с партнером по мобильным измерениям (MMP) и получите инструкции по соблюдению требований для получения доступа к атрибуции Meta для iOS 14+.** Проверьте свою конфигурацию в решении партнера по мобильным измерениям. Убедитесь, что соблюдены все требования партнера. Например, вам может потребоваться установить для своего приложения определенный переключатель на панели партнерского решения, чтобы получить доступ к атрибуции Meta для iOS 14+ (протоколу "Измерение на основе агрегированных данных о событиях").
  * **Параметр Advertiser Tracking Enabled вне допустимого диапазона.** Вы можете увидеть это сообщение, если возникли проблемы с количеством полученных нами событий с параметром Advertiser Tracking Enabled, значение которого — false или no. Если вы не установите параметр Advertiser Tracking Enabled, указывающий, что для отслеживания выбраны события на устройствах с iOS 14.5 и более поздними версиями, мы можем ограничить использование данных об этих событиях. [_Подробнее об этом параметре._](https://developers.facebook.com/docs/app-events/guides/advertising-tracking-enabled) Чтобы узнать больше, обратитесь к своему менеджеру аккаунта Meta.
  * **Ожидаем или подтверждаем информацию от вашей интеграции.** Информация о соответствии вашего события находится на рассмотрении. Обычно на получение информации от интеграции требуется 3 дня. Проверьте соответствие в Events Manager через 3 дня.
  * **Разные интеграции для множества типов событий.** Событие в приложении может не подходить для использования с целями по результативности "Максимальное число событий в приложении" и "Максимальная ценность конверсий" в кампаниям для продвижения приложения, если событие отправляется не через ту интеграцию, через которую мы получаем событие установки. Чтобы решить эту проблему, [_используйте одну интеграцию_](https://www.facebook.com/business/help/3215014965459549) для настройки события установки и события, которое хотите использовать для цели по результативности. Например, если для отправки события установки вы используете Facebook SDK для iOS, а для отправки события покупки — SDK от партнера по мобильным измерениям, то чтобы выполнить оптимизацию для события покупки, настройте его в конфигурации Facebook SDK для iOS. Чтобы обновить код, вам может понадобиться помощь разработчика. [_Подробнее о способах настройки и отправки событий в приложении._](https://www.facebook.com/business/help/235457266642587)
  * **ID установки не обнаружен или не дает достаточно информации.** Обратитесь к своему менеджеру аккаунта Meta, чтобы решить проблемы с подключением и соответствием требованиям. Вы также можете обратиться за помощью к своему партнеру по мобильным измерениям.
  * **Проблема с качеством интеграции.** Эта проблема возникает по одной из следующих причин:
    * Мы не получили никаких данных о событиях. Убедитесь, что событие в приложении правильно настроено и получено. Если событие в приложении не настроено, вы можете сделать это сейчас. [Подробнее о настройке событий в приложении](https://www.facebook.com/business/help/235457266642587).
    * Мы получили мало событий установки. Для оптимальной результативности кампании отправляйте все события конверсии.
    * Выбранная вами интеграция отправляет мало событий. Чтобы это исправить, [выберите другую интеграцию в Meta Events Manager](https://www.facebook.com/business/help/3215014965459549). После того как вы выберете интеграцию, нам потребуется до 7 дней для выполнения проверки на соответствие требованиям. В это время внесение изменений в кампанию может быть недоступно. Вы можете приостановить показ кампании до завершения проверки.
    * Если ваше событие в приложении по-прежнему не соответствует требованиям протокола "Измерение на основе агрегированных данных о событиях", обратитесь к своему менеджеру аккаунта Meta. чтобы решить проблемы с подключением и соответствием требованиям.
  * **Интеграция недоступна**. Ваш партнер по мобильным измерениям не внедрил протокол "Измерение на основе агрегированных данных о событиях". Свяжитесь с партнером и запросите интеграцию протокола. Если партнер не планирует интегрировать протокол "Измерение на основе агрегированных данных о событиях", вы можете использовать Facebook SDK для iOS. Вы или ваш разработчик можете воспользоваться [руководством по настройке Facebook SDK для iOS](https://developers.facebook.com/docs/app-events/guides/aggregated-event-measurement/#integration-with-the-facebook-sdk).
  * **Версия данных IP не подходит для конфигурации.** Ваша интеграция не отправляет IP-адреса или передает непоследовательные данных о них. Это может повлиять на качество данных. Чтобы узнать больше, обратитесь к своему менеджеру аккаунта Meta.
  * **Множественные интеграции для одного и того же события.** Эта проблема появляется, когда событие в приложении настроено в нескольких интеграциях. Из-за этого система регистрирует его в отчетах несколько раз. Также эта проблема может возникать, если событие в приложении отправляется не через ту интеграцию, через которую мы получаем события установки. Чтобы это исправить, выберите правильную интеграцию для отправки событий в Meta. Мы не будем регистрировать события из выбранной интеграции несколько раз. Это значит, что результаты кампаний, которые используют невыбранные интеграции, могут быть нарушены. Поэтому для отправки событий стоит выбрать наиболее важную для вашей компании интеграцию. После того как вы выберете интеграцию, нам потребуется до 7 дней для выполнения проверки на соответствие требованиям. В это время внесение изменений в кампанию может быть недоступно. Вы можете приостановить показ кампании до завершения проверки.
  * **Недостаточно сигналов за последние 30 дней**. Мы получали недостаточно сигналов от события в приложении за последние 30 дней. Убедитесь, что ваше приложение и события в нем настроены правильно и что события регистрируются в Meta Events Manager.
    * Если событие в приложении не настроено, вы можете сделать это сейчас. [Подробнее о настройке событий в приложении](https://www.facebook.com/business/help/235457266642587).
    * Если вы настроили событие в приложении, но оно не соответствует требованиям, воспользуйтесь инструментом тестирования событий. С его помощью вы сможете проверить правильность настройки и устранить проблемы. Подробнее о том, как использовать инструмент тестирования событий для [Facebook SDK, App Events API, интеграций партнеров по мобильным измерениям](https://www.facebook.com/business/help/2040882565969969) или [Conversions API](https://www.facebook.com/business/help/1624255387706033).
  * **Устаревший Facebook SDK для iOS**. Вы используете устаревшую версию Facebook SDK для iOS. Обновите Facebook SDK для iOS до версии 16.0.0 или более поздней. Вы или ваш разработчик можете воспользоваться [руководством по Facebook SDK для iOS](https://developers.facebook.com/docs/ios/upgrade-guide/) на сайте Meta for Developers.
  * **Устаревший SDK партнера по мобильным измерениям**. Ваш партнер внедрил протокол "Измерение на основе агрегированных данных о событиях", однако вы не установили последнюю версию партнерского SDK. Интегрируйте свое приложение с последней версией SDK партнера и обратитесь к нему за дополнительной информацией. Если ваше событие в приложении по-прежнему не отвечает требованиям, вы можете использовать Facebook SDK для iOS. Вы или ваш разработчик можете воспользоваться [руководством по настройке Facebook SDK для iOS](https://developers.facebook.com/docs/app-events/guides/aggregated-event-measurement/#integration-with-the-facebook-sdk).
  * **Неподтвержденная универсальная ссылка** и **Неподтвержденная глубокая ссылка**. Вы не подтвердили свою глубокую ссылку. Подтверждать глубокую ссылку обязательно, если вы используете Facebook SDK для iOS и хотите запустить кампанию с целью "Вовлеченность", "Продажи" или "Лиды". Для кампаний с целью "Продвижение приложения" подтверждение глубокой ссылки не требуется. Подробнее о том, [как подтверждать глубокие ссылки](https://www.facebook.com/business/help/734058628027282) и [использовать универсальные ссылки](https://www.facebook.com/business/help/1627795160837067).
