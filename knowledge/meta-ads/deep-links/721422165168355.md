# Об измерении на основе агрегированных данных Meta о событиях
![](https://static.xx.fbcdn.net/mci_ab/public/cms/?ab_b=e&ab_page=CMS&ab_entry=438223955063449&version=1666716831&transcode_extension=webp)
С 9 октября 2024 года мы отправляем отчетность по Измерениям на основе агрегированных данных о событиях (AEM) партнерам по мобильным измерениям (MMP), чтобы создать комплексное представление о результативности мер по привлечению новых пользователей на платформах MMP, включенных в отчетность. Прежде чем использовать данные для персонализации показываемой нами рекламы или измерения ее результативности, мы по-прежнему применяем методы защиты конфиденциальности, например удаление идентификаторов, добавление дифференциальной конфиденциальности, агрегацию данных пользователей. Мы сделали эту функцию доступной для всех партнеров MMP. Время интеграции может отличаться.
Измерение на основе агрегированных данных о событиях — это протокол Meta для измерения веб-событий и событий в приложении, полученных от пользователей устройств с iOS 14 и более поздними версиями. Прежде чем использовать данные для персонализации показываемой нами рекламы и измерения ее результативности, Измерение на основе агрегированных данных о событиях применяет методы защиты конфиденциальности, например удаление идентификаторов, добавление дифференциальной конфиденциальности, агрегацию данных пользователей.
Мы постепенно обновляем эту функцию для веб-событий и событий в приложении. Ниже приведена более подробная информация.

### Измерение на основе агрегированных данных о веб-событиях
Протокол "Измерение на основе агрегированных данных о событиях" позволяет нам компенсировать ограниченное измерение конверсий на сайте, выполняемых пользователями устройств с iOS 14.5 и более поздними версиями.
## Изменения в кампаниях для конверсий на сайте
Мы внесли изменения в кампании для конверсий на сайте. Теперь, чтобы ваши события обрабатывались через протокол "Измерение на основе агрегированных данных о событиях", не нужно выполнять никаких действий.
Получив доступ, вы увидите следующие изменения:
  * Больше не нужно выбирать 8 приоритетных событий конверсий на домен, чтобы выполнить оптимизацию для веб-конверсий. Также теперь не нужно включать группы значений ценности, чтобы использовать оптимизацию для ценности.
  * Вкладка "Измерение на основе агрегированных данных о событиях" в Meta Events Manager удалена, поскольку больше не нужно настраивать веб-события.
  * Не требуется подтверждать домены сайтов для целей, связанных с настройкой событий. Однако может понадобиться подтвердить домен по другим причинам. Подробнее о том, [в каких случаях нужно подтверждать домен](https://www.facebook.com/business/help/245311299870862).
  * При создании кампании в Ads Manager не нужно выбирать домен конверсии.


Протокол "Измерение на основе агрегированных данных о событиях" позволяет обеспечить конфиденциальность персональных данных, которые используются при создании отчетов о конверсиях и при оптимизации рекламы. Мы будем совершенствовать этот протокол, чтобы помочь рекламодателям обеспечивать конфиденциальность данных потребителей.

### Измерение на основе агрегированных данных о событиях в приложении
Протокол "Измерение на основе агрегированных данных о событиях" сейчас доступен для кампаний с местом назначения "Мобильное приложение" и целью "Продажи" (с включенным или выключенным каталогом), "Лиды" и "Вовлеченность".
## Изменения в кампаниях для приложений
Мы постепенно обновляем функции в кампаниях для приложений. Получив доступ, вы увидите следующие изменения:
  * **Протокол "Измерение на основе агрегированных данных о событиях" теперь также применяется для кампаний с целью "Продвижение приложения"**. Это значит, что вы сможете запускать кампании для продвижения приложения, не настраивая события в приложении для SKAdNetwork от Apple, а для оптимизации показа рекламы будет доступно больше событий в приложении. Если вы хотите просматривать результативность на разных платформах, то можете продолжить настраивать события для SKAdNetwork.
  * **Кампании для iOS 14+, в которых как метод атрибутирования используется протокол "Измерение на основе агрегированных данных о событиях", получают отчетность практически в режиме реального времени**. Так вы сможете быстрее скорректировать свою рекламную стратегию. [Подробнее об отчетности о событиях в приложении, в которой используется протокол "Измерение на основе агрегированных данных"](https://www.facebook.com/business/help/3499572073626051).
  * **Кампаниям для iOS 14+, в которых как метод атрибутирования используется протокол "Измерение на основе агрегированных данных о событиях", доступна отчетность "1 день после клика" и "7 дней после клика"** (для кампаний с оптимизацией для событий в приложении и оптимизацией для ценности).
  * **Кампаниям для iOS 14+, в которых как метод атрибутирования используется протокол "Измерение на основе агрегированных данных о событиях", доступны преимущества показа**. В частности, поддержка окон атрибуции "7 дней после просмотра" и показ на ресурсах Audience Network (для кампаний с оптимизацией для событий в приложении и оптимизацией для ценности). [Подробнее об оптимизации для событий в приложении](https://www.facebook.com/business/help/2308889442692949) и [оптимизации для ценности](https://www.facebook.com/business/help/296463804090290).


Вам не нужно ничего делать, чтобы получить доступ к этим изменениям. Однако могут потребоваться действия, чтобы привести ваше приложение в соответствие с требованиями. Подробнее о том, [как проверить приложение на соответствие требованиям в Events Manager](https://www.facebook.com/business/help/617021863580582).
**Примечание**. Данные о событиях, которые отправляются в Meta через Conversions API, также могут обрабатываться с учетом ограничений, определенных протоколом "Измерение на основе агрегированных данных о событиях".
