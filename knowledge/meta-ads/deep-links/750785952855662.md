# О наборах данных в Meta Events Manager
![](https://static.xx.fbcdn.net/mci_ab/public/cms/?ab_b=e&ab_page=CMS&ab_entry=438223955063449&version=1666716831&transcode_extension=webp)
Мы постепенно вносим [изменения в показ данных о событиях в Meta Events Manager](https://www.facebook.com/business/help/768703235046938). События на сайте, в приложении и события переписки можно будет просматривать в одном месте. Это называется набором данных. Эта статья будет вам полезна, если у вас есть доступ к новой версии наборов данных. 
Наборы данных позволяют подключать данные о событиях из разных источников (например, с вашего сайта, из мобильного приложения, обычного магазина или бизнес-чатов) и управлять ими в одном месте.
## Данные о событиях
События — это действия, которые люди выполняют в представительствах вашей компании, например на сайте, в мобильном приложении, обычном магазине или бизнес-чатах. Вы можете отправлять нам данные из этих источников с помощью таких интеграций, как пиксель Meta, Conversions API и Facebook SDK для iOS или Android. У каждой интеграции есть свой идентификатор, под которым показываются соответствующие события в Meta Events Manager (например, идентификатор пикселя, приложения или офлайн-события).
## Принцип работы наборов данных
Теперь вы можете подключить данные о событиях на сайте, в приложении, офлайн и в переписке с помощью набора данных. Создание набора данных не повлияет на оптимизацию кампаний и отчетность в Ads Manager.
Создав набор данных, вы получите его идентификатор, который сможете использовать для настройки интеграций и просмотра событий в Events Manager. Вы можете использовать идентификатор набора данных так же, как и идентификатор пикселя, например для настройки пикселя Meta или API Conversions. Если у вас есть приложение, его идентификатор не изменится и будет связан с вашим идентификатором набора данных.
После создания нового набора данных:
  * Идентификатор набора данных будет совпадать с идентификатором текущего пикселя, если пиксель связан с вашим набором данных. Ниже приведены примеры ситуаций, когда текущий пиксель связан с вашим набором данных.
    * Ваш пиксель был автоматически преобразован в набор данных.
    * Вы [создали набор данных в существующем мобильном приложении и выбрали ](https://www.facebook.com/business/help/5818684664831465)[вариант ](https://www.facebook.com/business/help/5818684664831465)[**Создать из ID пикселя**](https://www.facebook.com/business/help/5818684664831465).
  * Вы получите новый идентификатор набора данных, если текущий пиксель не связан с вашим набором данных. Ниже приведены примеры ситуаций, когда текущий пиксель не связан с вашим набором данных.
    * Ваше приложение было автоматически преобразовано в набор данных.
    * Вы [создали набор данных в существующем мобильном приложении](https://www.facebook.com/business/help/5818684664831465) и выбрали [вариант](https://www.facebook.com/business/help/5818684664831465)[**Создать новый**](https://www.facebook.com/business/help/5818684664831465).
    * Вы [создали набор данных для подключения офлайн-событий](https://www.facebook.com/business/help/5818684664831465).


Наборы данных могут показывать данные о событиях, полученных от любой из настроенных вами интеграций:
  * Пиксель Meta (события на сайте).
  * Conversions API (события на сайте, в приложении, офлайн-события и события переписки).
  * Facebook SDK для iOS или Android (события в приложении).
  * Решения партнеров по мобильным измерениям (события в приложении).
  * API App Events (события в приложении).
  * Загрузка файла в формате CSV (офлайн-события).


Чтобы подключить события в приложении к набору данных, сначала нужно привязать к нему идентификатор вашего мобильного приложения. При необходимости вы сможете [связать идентификатор приложения с другим набором данных](https://www.facebook.com/business/help/2499747716861514).
## Преимущества наборов данных
Наборы данных позволяют:
  * **Просматривать действия клиентов на основе потребностей компании независимо от источника**. Вместо отдельных идентификаторов для каждой интеграции у вас будет единый идентификатор набора данных, представляющий данные о событиях на сайте, в приложении, офлайн и в переписке. Так вы сможете отслеживать события и управлять настройками в одном месте.
  * **Отказаться от создания и поддержки нескольких интеграций API**. Раньше Conversions API использовался для прямого подключения вашего сайта к Meta, а App Events API, Offline Conversions API и Messaging Events API — для подключения данных о событиях в вашем мобильном приложении, офлайн и в переписке. Теперь, создав набор данных и привязав мобильное приложение, вы сможете напрямую подключить данные на сайте, в приложении, офлайн и в переписке с помощью Conversions API. Таким образом, вам потребуется управлять только одной интеграцией API с Meta.

## Будьте в курсе новостей от Meta for Business
Укажите электронный адрес, на который вы хотите получать новости о мероприятиях и продуктах, а также другую информацию от Meta for Business. 
Отправляя эту форму, вы соглашаетесь получать по электронной почте маркетинговые материалы от Meta, в том числе новости, информацию о мероприятиях, обновления и специальные предложения. Вы можете отказаться от рассылки в любой момент. Для этого нужно, например, нажать ссылку для отмены подписки в нашем электронном письме.
