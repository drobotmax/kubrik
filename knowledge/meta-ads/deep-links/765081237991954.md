# О качестве сопоставления событий
![](https://static.xx.fbcdn.net/mci_ab/public/cms/?ab_b=e&ab_page=CMS&ab_entry=438223955063449&version=1666716831&transcode_extension=webp)
Качество сопоставления оценивается для событий на сайте, которые отправляются через Conversions API и используют параметр источника действия **Сайт**.
Качество сопоставления событий указывает на то, насколько параметры информации о клиентах, полученные вместе с событиями с вашего сервера, полезны для сопоставления этих событий с аккаунтами на платформах Meta.
Сопоставление событий помогает показывать рекламу людям, которые с большей вероятностью выполнят нужное вам действие, а затем атрибутировать эти действия вашей рекламе. Более качественное сопоставление событий означает, что события с большей вероятностью будут сопоставлены с аккаунтами на платформах Meta. Сопоставление событий помогает атрибутировать результаты вашей рекламе и показывать ее людям, которые с наибольшей вероятностью выполнят конверсию. Это может улучшить результативность рекламы и снизить цену за действие. 
Подробнее о том, [как проверить качество сопоставления событий в Meta Events Manager](https://www.facebook.com/business/help/1541268312717919).
## Как определяется качество сопоставления событий?
Meta ставит оценку от 0 до 10 с учетом качества информации о клиентах, которая отправляется для определенного события сервера, и процента экземпляров события, сопоставленных с аккаунтами на платформах Meta.
Оценки качества сопоставления событий выставляются для всех стандартных и специально настроенных событий на сайте, например событий просмотра страницы, добавления в корзину и покупки. Для оценок используется информация за последние 48 часов, поэтому для их актуальности важно регулярно отправлять события.
## Как параметры информации о клиентах помогают улучшить сопоставление событий?
Чтобы улучшить долю сопоставления событий с аккаунтами на платформах Meta, отправляйте по возможности несколько параметров информации о клиентах. Более качественное сопоставление событий может увеличить [_количество дополнительных зарегистрированных конверсий_](https://www.facebook.com/business/help/453888373437795). В Events Manager приводятся рекомендации по улучшению сопоставления событий.
