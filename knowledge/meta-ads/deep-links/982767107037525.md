# О местах назначения данных
Место назначения данных — это конечная точка, куда вы отправляете данные со своего облачного сервера Signals Gateway. Поступившие на сервер данные можно направлять в несколько мест назначения, чтобы оптимизировать передачу данных и расширить возможности аналитики.
##  Типы мест назначения данных 
  * **Пользовательский коннектор**. Отправляйте события в уникальную конечную точку, например во внутреннюю систему, хранилище данных или на аналитическую платформу. Это предоставит вам гибкость для персонализации передачи данных в соответствии с вашими конкретными бизнес-потребностями.
  * **Коннектор Meta**. Используйте плагин Meta Conversions API, чтобы передавать данные в экосистему Meta для бесшовной интеграции с рекламными инструментами Meta.
  * **Коннектор Google BigQuery**. Отправляйте события в хранилище Google BigQuery, предоставляющее мощные возможности для масштабной обработки, хранения и аналитики данных


**Пример использования**. Предположим, что вы хотите проанализировать данные о покупках клиентов. Отправив эти данные в Google BigQuery, вы сможете выполнить расширенную аналитику, чтобы получить представление о тенденциях покупок и поведении клиентов.
##  Настройка мест назначения данных 
Чтобы настроить новые места назначения, откройте страницу **Data pipelines** ("Потоки данных") в своем аккаунте Signals Gateway. Настройте каждый тип места назначения согласно приведенным ниже инструкциям.
##  Конфигурация пользовательского коннектора 
  1. Нажмите **Create data pipeline** ("Создать поток данных") в своем аккаунте Signals Gateway.
  2. Выберите источники данных.
  3. Выберите место назначения данных **Customized connector** ("Пользовательский коннектор").
  4. Настройте раздел **Connector Settings** ("Настройки коннектора").


  * Введите URL пользовательской конечной точки.
  * Настройте необходимые данные для аутентификации.
  * Сопоставьте поля данных из вашего экземпляра Signals Gateway с полями, ожидаемыми вашей пользовательской конечной точкой.
  * Отправьте тестовое событие и убедитесь, что соединение успешно установлено.


##  Плагины 
В Signals Gateway уже установлены плагины для Meta Conversions API и Google BigQuery. Однако данные не будут передаваться в Meta или Google BigQuery, пока вы не создадите поток данных с этими местами назначения.
##  Установка и подтверждение 
  1. В меню своего аккаунта Signals Gateway выберите **Плагины**.
  2. Убедитесь, что для плагина задан статус **Installed** ("Установлено").
  3. После установки выберите поток **Meta Connector** ("Коннектор Meta") или **Conversions API Gateway** ("Шлюз Conversions API") в соответствии с вашим способом передачи данных.


**Примечание**. Плагин установлен по умолчанию. Однако передача данных через Signals Gateway в Meta начнется, только когда вы создадите поток данных и выберете для него место назначения "Meta".
##  Устранение проблем 
Если при включении плагина возникнут проблемы, попробуйте:
  * Подтвердить разрешения администратора.
  * Проверить наличие неустановленных обновлений или уведомлений, которые могут влиять на работу плагина.


##  Плагин Meta Conversions API 
Плагин Meta Conversions API позволяет передавать данные в Meta Conversions API как в место назначения. Вы можете использовать:
  1. **Поток Signals Gateway**. В таком случае информация собирается из источника данных Signals Gateway, например с помощью Signals Gateway Pixel.
  2. **Поток шлюза Conversion API**. В таком случае данные собираются с помощью пикселя Meta.


##  Плагин Google BigQuery 
Коннектор Google BigQuery позволяет отправлять данные из Signals Gateway в Google BigQuery для расширенного анализа и отчетности.
##  Настройка плагина Google BigQuery 
  1. **Выберите потоки данных**. В своем аккаунте Signals Gateway выберите **Data pipelines** ("Потоки данных").
  2. **Укажите место назначения**. Выберите **Google BigQuery**.
  3. **Следуйте документации**. Получите подробные инструкции по настройке в [документации для Google BigQuery](https://l.facebook.com/l.php?u=https%3A%2F%2Fcloud.google.com%2Fbigquery%2Fdocs&h=AT4Uue2GhCLmmK8htSDa_CpRcVVLNsgKS4IO6Y0ScArLaARSMqmgY6gH42hPWwS8hEmzpMIqpevTZFFWrMcvfrY5KNrtNtKKO6EwwLfqp0n6BEe8YVAuuDd5e4J5kp1I38I_Ohps9ff79A).
  4. **Протестируйте соединение**. Убедитесь, что данные корректно отправляются в BigQuery.


##  Глоссарий 
  * **Интерфейс программирования приложений (API)** — набор правил, который позволяет программам взаимодействовать друг с другом.
  * **Конечная точка** — URL, по которому API получает запросы.
  * **Пиксель** — небольшой фрагмент кода на сайте, собирающий данные.
  * **Схема** — формально описанная структура системы базы данных.


Более подробные инструкции см. в документации для [Google BigQuery](https://l.facebook.com/l.php?u=https%3A%2F%2Fcloud.google.com%2Fbigquery%2Fdocs&h=AT5UTYZOLT645nazCpvbun6rM0UGdarRyayR_rfc6-9UNE9vXzfYtiC4cPB4p2v-s5j4TjwjpgFmx8xAeCiCPZcZ1osLN0HR2_aH2QGGI7C7hast58LZG9ravYHWkJIG_38nLzJ_HptIeg) и [разработчиков](https://developers.facebook.com/docs/marketing-api/gateway-products/signals-gateway/data-destination-management).
