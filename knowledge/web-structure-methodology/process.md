# Эталонный процесс разработки сайта

## Этапы и артефакты

| Этап | Артефакт | Формат | Роль | Длительность |
|------|----------|--------|------|--------------|
| 1. Паспорт | Паспорт проекта | Miro / Google Doc | Проектировщик + клиент | 1 созвон + доработка |
| 2. Структура | Структура сайта | Miro (карта страниц + блоки) | Проектировщик -> внутр. ревью -> клиент | 1-2 созвона |
| 3. Прототип | Wireframes | Excalidraw / Figma | Проектировщик + клиент | ~3 недели (по UPH) |
| 4. ТЗ | Техническое задание | Google Doc | Проектировщик | ~1 неделя |
| 5. Дизайн | UI | Figma | Дизайнер + клиент | отдельный этап |
| 6. Разработка | Работающий сайт | CMS / код | Разработчик | отдельный этап |

## 1. Паспорт проекта

Фиксирует "зачем" до "как". Без паспорта структура делается вслепую.

**Что должно быть:**
- Бизнес-цели проекта (не "сайт", а "лидогенерация / расширение рынков / узнаваемость / лояльность")
- ЦА: сегменты, JTBD, боли
- Позиционирование (УТП, tone of voice)
- Анализ конкурентов (что делают, в чём у нас преимущество)
- Ограничения: CMS, бюджет, сроки, интеграции
- Бренд-элементы: лого, палитра, шрифты (если есть)

**Замечание из кейса UPH:** созвон по паспорту занимает много времени, и часто не успеваешь обсудить все спорные вопросы. Это нормально - недостающие куски фиксируются как "на обсуждение" и закрываются позже.

## 2. Структура

Карта всех страниц сайта + список блоков на каждой + переходы между страницами.

**Что должно быть:**
- Шапка: логотип, меню, контакты, price (опционально), языковые версии, поиск, кнопка "получить консультацию"
- Главная: промо + каталог + готовые решения + proof + форма
- Под-категория (если каталог): промо + продуктовая подборка + тех. детали
- Карточка товара/услуги: галерея + описание + преимущества + форма + документы + характеристики
- Внутренние страницы под бизнес-процессы (дилеры, партнёры, госзакупки, сотрудничество)
- О компании, Новости, Контакты, Политика
- Футер: дубль меню, контакты, поиск

**Методология принятия решений:**
- Если есть несколько вариантов организации каталога (по технологии / отрасли / виду) - сделать **все варианты в Miro** и обсуждать с клиентом.
- Сверить группировку каталога с **ЦРМ и 1С** клиента - иначе придётся переделывать.
- Для каждого "серого" блока фиксировать **вопрос клиенту** прямо в структуре ("здесь пусто - нужна информация: какие компании интересны").

**Внутренняя проверка перед клиентом:**
Обязательный созвон с коллегой (в UPH: Артём -> Настя). 5-10 минут walkthrough ловят 80% проблем до клиента.

## 3. Прототип

Wireframes всех ключевых экранов (главная, каталог, под-категория, карточка, отраслевая, контакты + все специальные страницы).

**В кейсе UPH:** Excalidraw, ~157 текстовых элементов, детально до уровня "поле ввода имени" / "кнопка отправить" / "tg wa".

**Цена референс:** 40-60К руб.

## 4. ТЗ

Документ для разработчика: стек, CMS, интеграции, SEO-требования, админка, формы и их назначение, интеграция с CRM, аналитика, мобильная версия, производительность.

**Цена референс:** 30-40К руб.

## 5-6. Дизайн и разработка

За рамками методологии структуры. Тут включаются дизайнер + разработчик.

## Гейт-поинты (checkpoints)

- **После паспорта:** согласован ли бизнес-смысл проекта? Есть ли измеримые KPI?
- **После структуры:** все ли страницы покрыты? Сверено ли с ЦРМ/1С? Закрыты ли "пустоты"?
- **После прототипа:** клиент "видит" сайт? Все состояния форм покрыты (отправка / успех / ошибка)?
- **После ТЗ:** разработчик понимает, что делать, без дополнительных вопросов?

## Анти-паттерны (замеченные в кейсе)

1. **Прыгать в прототип без структуры.** Получается каша и много переделок.
2. **Не сверять каталог с ЦРМ.** Клиент потом говорит "у нас в 1С по-другому" - переделка всего каталога.
3. **Показывать клиенту один вариант.** Когда есть неоднозначность - показать 2-3 и обсудить. Иначе спорим впустую.
4. **"Ничего не найдено" как тупик.** Если поиск пустой - форма обратной связи, а не 404.
5. **Price без процесса актуализации.** Устаревший прайс хуже, чем его отсутствие.
6. **Показывать текущих дилеров потенциальным.** Демотивирует: "рынок уже захвачен".
